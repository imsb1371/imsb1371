The first user study I ran in graduate school was a disaster. Not in the catastrophic sense — participants were fine, the IRB was happy — but in the sense that I learned almost nothing useful from it. The task was too constrained. The measures were borrowed from papers without understanding their assumptions.

Here is what I do differently now.

## Start with the Question, Not the Method

The most common mistake I see in student proposals is method-first thinking: "I want to run a controlled experiment." The method should follow the question, not precede it.

If you want to know *whether* something works, you probably need a controlled experiment. If you want to know *why* or *how* something works, you probably need qualitative methods.

## Pre-Registration Matters

I now pre-register all quantitative studies, including the analysis plan, before data collection begins. This forces rigor at the design stage and dramatically reduces the temptation to HARKing (Hypothesizing After Results are Known).

## Pilot Everything Twice

One pilot is never enough. The first pilot tests the logistics. The second tests whether the task elicits the behaviors you actually want to study. I've killed or substantially redesigned more than a few studies based on second pilots — and my publications are better for it.
